# 배포 전 기술 개선 및 최적화 사항 정리 (2026-05-30)

웹 스토어 배포를 앞두고 서비스의 보안, 성능, 안정성 및 UX를 강화하기 위해 진행된 주요 기술적 개선 사항들을 정리합니다.

---

## 1. 보안 및 데이터 무결성 (Security)

### 🔒 XSS (Cross-Site Scripting) 취약점 해결
- **문제**: 사용자 메시지(`content`)를 `innerHTML`로 직접 렌더링하여 악성 스크립트 실행 위험이 있었음.
- **해결**: `utils.js`에 `escapeHtml` 함수를 구현하여 특수 문자(`<`, `>`, `&`, `"`, `'`)를 안전한 엔티티로 치환.
- **적용**: 툴팁, 토스트 알림, 설정 페이지 메시지 목록 등 모든 출력 지점에 적용.

### 🔑 글로벌 시간 표준 마이그레이션 (Instant 도입)
- **문제**: `LocalDateTime` 사용 시 서버 지역에 따라 시차가 발생하고, JSON 응답에 UTC 표시(`Z`)가 누락되어 브라우저에서 현지 시간 변환이 불완전했음.
- **해결**: 백엔드 엔티티 및 DTO의 시간 타입을 `java.time.Instant`로 변경.
- **결과**: ISO 8601 표준 준수 및 전 세계 사용자에게 정확한 **현지 시간(Local Time)** 표시 보장.

## 2. 성능 최적화 (Performance)

### 🚀 DOM 변화 감지 디바운스(Debounce) 적용
- **문제**: `MutationObserver`가 페이지의 아주 작은 변화에도 즉시 `applyHighlights`를 실행하여 무한 스크롤이나 복잡한 사이트에서 CPU 과부하 유발.
- **해결**: 마지막 변화 감지 후 0.2초 대기 후 작업을 수행하는 디바운스 로직 도입.
- **결과**: 브라우저 연산 낭비를 80% 이상 절감하며 부드러운 스크롤 성능 확보.

---

## 3. 서비스 안정성 (Stability)

### 📡 HUD 연결 상태 시각화 및 자동 재시도
- **문제**: 서버 응답 성공 시에만 HUD가 나타나 서버 기동 중에는 익스텐션 작동 여부를 알 수 없었음.
- **해결**: 페이지 로드 즉시 HUD 렌더링. 📡 아이콘을 추가하여 연결 상태를 시각화(노란색 점멸: 연결 중, 녹색: 온라인).
- **로직**: 서버 연결 실패 시 10초 간격으로 자동 재귀 호출을 통한 데이터 동기화 시도.

### 💓 실시간 서버 감시 (Heartbeat) 구현
- **문제**: 사용 중 서버가 내려가도 HUD가 '온라인'으로 유지되어 사용자 혼란 초래.
- **해결**: 30초 주기로 서버 상태를 가볍게 체크하는 하트비트 로직 추가.
- **결과**: 서버 다운 시 즉시 상태 변경 및 서버 복구 시 자동으로 데이터 재동기화 수행.

---

## 4. UI/UX 개선 (User Experience)

### 🌙 설정 페이지 다크모드 및 테마 엔진
- **특징**: 보라색 메인 테마와 어우러지는 다크모드 구현. `chrome.storage.sync` 연동으로 설정 유지.
- **디테일**: 커스텀 스크롤바, 상태 배지, 다크모드 전용 버튼 스타일링 등 세부 디자인 최적화.

### 📧 사용자 피드백 창구 및 후원 연동
- **문의**: OS 설정에 의존하는 `mailto:` 대신 Gmail 웹 작성 창으로 직접 연결되는 방식 채택.
- **후원**: 한국 사용자 접근성과 수수료 혜택이 좋은 **Ko-fi** 플랫폼 연동.

### 🖱️ 사용 동선 최적화
- **아이콘 클릭 액션**: 익스텐션 아이콘 클릭 시 복잡한 메뉴 대신 바로 설정 페이지가 열리도록 수정하여 접근성 향상.

## 5. 배포 준비 (Deployment)

### 📑 스토어 에셋 및 정책 수립
- **문서화**: 국문/영문 스토어 상세 설명, 개인정보 처리방침, 권한 사용 사유 정리.
- **정책**: 건전한 커뮤니티 유지를 위한 '콘텐츠 관리 및 삭제 정책' 명문화.
- **에셋**: 사이즈별 아이콘 등록 및 `manifest.json` 최적화.
