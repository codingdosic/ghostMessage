# 백엔드 유지보수 및 코드 반영 가이드 (Backend Maintenance Guide)

이 문서는 백엔드 코드 수정 시 로컬 개발 환경에서부터 오라클 클라우드(1GB RAM) 서버 반영까지의 **전체 워크플로우**를 다룹니다.

---

## 1. 전략적 워크플로우: 코드에서 배포까지

### 면접 대비: "운영 중인 서버의 코드를 어떻게 업데이트하나요?"
> **답변 핵심**: "소스 코드의 형상 관리는 **Git**을 중심으로 진행하며, 배포는 **Docker 컨테이너** 단위를 사용합니다. 특히 1GB RAM의 저사양 환경이므로, 서버 내 빌드 시 발생할 수 있는 메모리 부족 현상을 방지하기 위해 **SWAP 활용 및 단계적 컨테이너 재시작 전략**을 사용합니다. 최종적으로는 GitHub Actions를 통한 CI/CD 파이프라인으로 이 과정을 자동화하여 휴먼 에러를 방지하는 것을 목표로 합니다."

---

## 2. 단계별 수정 및 반영 절차

### Step 1: 로컬 코드 수정 및 검증
1.  `ghostMessage/src/main/java/...` 내의 코드를 수정합니다.
2.  로컬에서 `./gradlew bootRun` 또는 IDE를 통해 기능이 정상 작동하는지 확인합니다.
3.  **중요**: 새로운 라이브러리(의존성)를 추가했다면 `build.gradle`에 정확히 반영되었는지 확인합니다.

### Step 2: 소스 코드 동기화 (Git)
서버에 코드를 전달하기 위해 GitHub를 경유합니다.
```bash
git add .
git commit -m "feat: 특정 기능 추가 또는 버그 수정"
git push origin main
```

### Step 3: 서버 코드 업데이트 및 재빌드
오라클 서버 터미널에 접속하여 최신 코드를 불러오고 컨테이너를 갱신합니다.

1.  **최신 코드 Pull**:
    ```bash
    cd ~/app
    git pull origin main
    ```
2.  **컨테이너 재빌드 및 실행**:
    ```bash
    sudo docker-compose up -d --build app
    ```
    *   `--build`: 소스 코드 변경 사항을 반영하여 새로운 이미지를 만듭니다.
    *   `app`: DB와 Redis는 그대로 두고 **백엔드 앱 컨테이너만** 골라서 재빌드하여 자원 소모를 최소화합니다.

---

## 3. 1GB RAM 환경에서의 주의사항 (Troubleshooting)

### 빌드 중 서버 멈춤 현상 대응
1GB RAM 서버에서 `docker-compose up --build`를 실행하면 Gradle 빌드 과정에서 메모리 점유율이 급격히 올라가 서버가 멈출 수 있습니다.

*   **해결책 1**: 반드시 **SWAP 메모리(2GB)**가 활성화되어 있는지 사전에 확인합니다 (`free -h`).
*   **해결책 2**: 서버 내 빌드가 너무 힘들 경우, 로컬에서 빌드한 `.jar` 파일만 서버로 보내 `Dockerfile`에서 빌드 과정을 생략하는 방식으로 우회할 수 있습니다. (향후 GitHub Actions 도입 시 해결될 문제)

## 4. 기술 면접 예상 질문

**Q: 운영 중인 서버에서 코드를 수정할 때 가장 주의하는 점은 무엇인가요?**
**A**: "**서비스 가동 시간(Uptime)** 유지입니다. 백엔드 컨테이너를 재빌드하는 동안 일시적인 중단이 발생할 수 있는데, 이를 최소화하기 위해 DB나 캐시는 그대로 둔 상태에서 앱 컨테이너만 타겟팅하여 업데이트합니다. 또한, 배포 전 로컬 환경에서 충분한 테스트를 거쳐 '배포 후 롤백' 상황이 생기지 않도록 관리합니다."

**Q: 수동으로 'git pull'을 해서 배포하는 방식의 한계는 무엇인가요?**
**A**: "작업자가 서버에 직접 접속해야 하므로 보안상 위험이 있고, 실수로 명령어를 잘못 입력할 가능성이 있습니다. 또한 서버 사양이 낮을 경우 빌드 부하를 서버가 직접 감당해야 한다는 단점이 있습니다. 이를 극복하기 위해 빌드는 외부(GitHub Actions)에서 수행하고, 서버는 이미 빌드된 결과물(이미지)만 내려받아 실행하는 **CI/CD 자동화**가 반드시 필요합니다."

---
*이 가이드를 숙지하면 코드 수정이 두렵지 않은 안정적인 운영 환경을 유지할 수 있습니다.*
