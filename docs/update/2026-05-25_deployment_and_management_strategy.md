# GhostMessage 배포 및 운영 가이드

본 문서는 GhostMessage 프로젝트의 성공적인 상용 배포와 지속 가능한 운영을 위한 전략을 정리한 가이드입니다. 익스텐션의 웹 스토어 등록부터 백엔드 서버 관리, 그리고 프로젝트 유지를 위한 후원 방안까지 포함합니다.

---

## 1. 크롬 익스텐션 웹 스토어 배포 (Chrome Web Store)

익스텐션을 대중에게 공개하기 위한 정식 배포 절차입니다.

### 1.1 등록 준비
- **개발자 계정 등록:** 구글 개발자 대시보드에서 등록비($5, 일회성) 결제 필요.
- **아이콘 및 스크린샷:** 128x128(아이콘), 1280x800 또는 640x400(스크린샷 최소 1장) 준비.
- **프로필 정보:** 이름, 설명, 개인정보처리방침(Privacy Policy) URL.
- **Manifest V3 준수 확인:** 현재 프로젝트는 이미 V3 기반으로 작성되어 있어 적합합니다.

### 1.2 보안 검토 및 권한
- **최소 권한 원칙:** `activeTab`, `storage`, `contextMenus` 등 현재 사용 중인 권한의 필요성을 상세히 설명해야 합니다.
- **원격 코드 실행 금지:** 모든 로직은 패키지 내부에 포함되어야 합니다 (현재 구조 유지).
- **개인정보처리방침:** UUID 수집 목적과 데이터 보관 기간을 명시한 간단한 웹페이지가 필요합니다 (GitHub Pages 추천).

---

## 2. 백엔드 서버 및 데이터 관리

실제 운영 환경에서의 안정성을 확보하기 위한 관리 방안입니다.

### 2.1 서버 모니터링 및 로깅
- **Prometheus & Grafana:** Spring Boot Actuator를 연동하여 서버 리소스(CPU, Memory)와 API 호출 성공률을 시각화합니다.
- **Docker 로그 관리:** `docker logs --tail 100 ghost-app` 등을 정기적으로 확인하거나, 로그 파일을 외부 볼륨으로 추출하여 관리합니다.

### 2.2 데이터 백업 (Data Persistence)
- **PostgreSQL 백업:** `pg_dump`를 사용하여 매일 자정 DB 데이터를 SQL 파일로 백업하고, 이를 별도의 외부 스토리지나 클라우드(S3 등)로 전송하는 스크립트를 운용합니다.
- **Redis 관리:** 현재 `allkeys-lru` 정책이 설정되어 있으므로, 캐시 데이터가 유실되어도 DB에서 복구 가능한 구조인지 상시 확인합니다.

### 2.3 보안 강화
- **HTTPS 적용:** `Certbot(Let's Encrypt)`을 사용하여 API 서버에 SSL 인증서를 적용하고, 익스텐션의 `SERVER_URL`을 `https://`로 전환해야 합니다 (익스텐션 보안 정책상 필수 권장).
- **CORS 설정:** 현재 `origins = "*"`인 설정을 배포 시에는 특정 도메인이나 익스텐션 ID로 제한하여 보안을 강화합니다.

---

## 3. 후원 및 수익화 방안 (Monetization)

무료 서비스의 서버 비용 충당과 지속적인 개발을 위한 후원 연동 방안입니다.

### 3.1 후원 플랫폼 선정
- **Buy Me a Coffee / Ko-fi:** 수수료가 낮고 버튼/위젯을 익스텐션 UI에 삽입하기 매우 쉽습니다.
- **Patreon:** 정기 후원자에게 특별한 칭호(닉네임 색상 변경 등)를 제공할 경우 적합합니다.

### 3.2 익스텐션 내 UI 연동
- **Options 페이지:** 설정 화면 하단에 "개발자에게 커피 한 잔 선물하기" 버튼 배치.
- **Tooltip/HUD:** 메시지 확인 UI 구석에 작은 하트 아이콘이나 후원 링크 포함.
- **후원자 혜택 (비강제):** 
    - 메시지 작성 한도 증설 (예: 일일 10개 -> 30개).
    - 후원자 전용 닉네임 뱃지 제공 (서버 DB `User` 테이블에 `is_supporter` 필드 추가 필요).

---

## 4. 향후 운영 로드맵

1.  **Phase 1 (Beta):** 현재 상태에서 HTTPS 적용 및 소수 인원 테스트.
2.  **Phase 2 (Release):** 크롬 웹 스토어 등록 및 후원 링크 오픈.
3.  **Phase 3 (Expansion):** 다국어 지원(i18n) 및 신고 시스템(Abuse Report) 도입을 통한 커뮤니티 정화 기능 강화.

---

## 5. 결론
성공적인 운영을 위해서는 **안정적인 서버 관리(백업/모니터링)**와 **사용자와의 투명한 소통(개인정보방침/후원 혜택)**이 핵심입니다. 위 절차를 단계적으로 실행한다면 GhostMessage는 지속 가능한 서비스로 자리 잡을 수 있습니다.
